# Canon Snapshot (v0)

This is the **load-bearing, ratified** project context.

## D-0001 — Assist-only AI
**Decision:** AI may analyze, summarize, propose, and critique; it must never auto-enforce rules, moderation, or canon.

## D-0002 — Canon requires explicit ratification
**Decision:** Only explicit **Decision objects** are binding. Chat transcripts and summaries are evidence, not canon.

## D-0003 — Provenance + epistemic friction are core mechanics
**Decision:** Every serious claim should carry provenance (source references) and visible uncertainty when appropriate.

## D-0004 — Custom nukeCE chat interface (Option B)
**Decision:** Build a custom chat interface so all messages are captured natively into nukeCE (append-only transcripts + IDs + hashes).

## D-0005 — Context Packs are the session glue
**Decision:** Each session loads a Context Pack (canon snapshot + active campaign + module scope) to avoid drift and re-explain tax.

## D-0006 — Minimal schema + folder structure is canonical starting point
**Decision:** Use a minimal model (Conversation/Message/Artifact/Claim/Decision/ReferenceNode) and a separated folder tree (transcripts vs canon vs reference).
