# Minimal schema and folder structure

**Tags:** architecture, infrastructure

**Status:** reviewed

## Summary
The minimal model uses append-only messages as evidence and distinct objects for artifacts, claims, decisions, and reference nodes—mapped to a sane repo tree.

## Mechanics
This supports auditability, retrieval, and long-term continuity without bloating prompts.

## Non-goals
- This node is not canon by itself. Canon lives in `canon/decisions/*`.
- Do not treat chat summaries as binding without ratification.
